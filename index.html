Round One

Task Description:

In this challenging first round, participants are required to submit an Innovative Project Proposal that addresses a real-world problem or opportunity. Teams should meticulously outline their project concept, its relevance, and the proposed technical solution. The goal is to assess both the creativity of the idea and the team's ability to articulate a clear and compelling vision.

Submission Guidelines:

  • Identify a pressing real-world problem or an untapped opportunity.
  • Propose an innovative technological solution to address the identified challenge.
  • Provide a detailed project overview, including key features and functionalities.
  • Clearly define the target audience and potential impact of the proposed solution.
  • Outline the technology stack and tools that will be utilized in the project.

Evaluation Criteria:

  • Creativity and uniqueness of the project concept.
  • Clarity and coherence in presenting the problem and solution.
  • Feasibility and practicality of the proposed technical solution.
  • Demonstration of a deep understanding of the chosen problem domain.
  • Overall presentation and adherence to submission guidelines.

This task is designed to evaluate the strategic thinking, problem-solving skills, and vision of the participants. The top 10 teams with the most promising and innovative project proposals will advance to the second round of Innovate-a-Thon.

Round Two

Task Description:

In this advanced stage of the hackathon, selected teams are tasked with developing a fully integrated solution that addresses the problem or opportunity outlined in their Round One Innovative Project Proposal. The challenge is to bring their concept to life by coding a functional prototype that demonstrates the core features of their innovative solution.

Key Components:

  • Implementation: Code a working prototype of your proposed solution, ensuring it aligns with the outlined features and functionalities.
  • Integration: Seamlessly integrate various components of your solution to showcase a cohesive and interconnected system.
  • User Interface (UI) / User Experience (UX): Design an intuitive and user-friendly interface that enhances the overall user experience.
  • Functionality: Ensure that the core functionalities of your solution are operational and effectively address the identified problem or opportunity.
  • Technical Documentation: Provide comprehensive technical documentation that explains the architecture, data flow, and key algorithms used in your solution.
  • Presentation: Prepare a compelling presentation that effectively communicates the value proposition, innovation, and impact of your integrated solution.

Evaluation Criteria:

  • Functionality and Completeness: The extent to which the implemented solution aligns with the proposed features and functionalities.
  • Innovation: The creativity and uniqueness demonstrated in the technical approach and problem-solving.
  • Integration: The effectiveness of integrating different components into a seamless solution.
  • User Interface / User Experience: The quality of the designed interface and the overall user experience.
  • Technical Documentation: The clarity and completeness of the provided technical documentation.
  • Presentation: The ability to convey the significance and impact of the solution in a clear and engaging manner.

Teams will be evaluated based on these criteria to determine the winners of the Innovate-a-Thon. Good luck!

REQUIREMENTS

WHAT TO BUILD:

  • Submissions can consist of products with interfaces that can be demoed, or they can consist of contract code that can be effectively deployed on Futurenet or in standalone mode.
  • Your code repository must be public, and must include an open-source license.
  • All submissions must be created during the timeframe of the hackathon.

JUDGING CRITERIA

Functionality & Purpose
Are you solving an existing problem? How scalable is the application? Can it be used in other regions, or can it be used by more than one type of audience? Is it an application that would encourage a user to use it more than once for its benefits?

Documentation
Is the submission description complete and easy to read? Does the code have comments that explain what's going on? Project must include instructions on how others can contribute in README.md file on GitHub and must have a license.

Creativity
Project takes an original approach to a problem. Solution effectively alleviates, solves, or helps the problem it is designed to solve.

Design
Design is accessible, well thought out, demonstrates smart design choices, and is user-friendly.
